# Анализ исходного Vania_Sait.zip

Исходный архив содержал небольшой статический сайт ShioMI и backend на Node/Express.

## Найденные проблемы

1. В репозитории лежали `.git` и `node_modules`, из-за чего архив был раздут и плохо переносился.
2. Backend был написан на Express, а не на Django.
3. Использовалась внешняя MySQL-база с жёстко прописанными логином, паролем, IP и названием базы прямо в `server.js`.
4. Steam API key и session secret были захардкожены в коде.
5. Frontend был написан как один `index.html`, один `profile.html` и один общий `style.css`, без компонентов и типизации.
6. В `profile.html` проверялось поле `data.steamId`, но Express backend его не возвращал — из-за этого профиль мог не открываться после авторизации.
7. В аккордеоне правил JavaScript переключал класс `active`, а CSS ожидал класс `open`; раскрытие правил работало некорректно.
8. Logout-ссылка в интерфейсе указывала на `/api/auth/logout`, но такого маршрута в Express backend не было.
9. Рулетка кейса имела HTML/CSS-разметку, но не имела полноценной рабочей backend-логики выдачи наград.
10. Не было Docker Compose, Nginx, Gunicorn, PostgreSQL, Django Admin, media/static настройки, backup-скриптов, Sentry и Uptime Kuma.

## Что невозможно восстановить автоматически

Содержимое исходной внешней MySQL-базы не было внутри архива. Поэтому реальные игровые записи `lr_base`, `vip_users`, `admin_users`, `site_users` не перенесены. Вместо этого создана безопасная PostgreSQL-модель в Django, которую можно наполнить через Django Admin или миграцию/импорт.
